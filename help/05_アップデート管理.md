# アップデート管理

GitHub private リポジトリを使って、アプリの自動更新を行います。  
管理者が `git push` するだけで、従業員のアプリが次回起動時に自動で差分を取得・再起動します。

---

## 仕組み

1. **管理者**がコードを修正して `git push`（manifest.json も一緒にコミット）
2. **従業員**がアプリを起動すると、GitHub API で `manifest.json` を取得
3. バージョンが新しければ「アップデートがあります」ダイアログを表示
4. 「はい」を押すと変更ファイルのみ GitHub からダウンロードして上書き
5. アプリが自動で再起動

---

## 管理者の設定手順（初回のみ）

### ステップ 1 — GitHub に private リポジトリを作成

1. GitHub.com でリポジトリを新規作成（Private）
2. 以下のコマンドでコードをプッシュ:

```
cd Automation-Userlist
git init
git add .
git commit -m "Initial release v2.0.0"
git remote add origin https://github.com/<アカウント>/<リポジトリ名>.git
git push -u origin main
```

### ステップ 2 — Fine-grained PAT を作成（従業員配布用）

1. GitHub.com → 右上アイコン → Settings
2. 左メニュー最下部 → Developer settings → Personal access tokens → Fine-grained tokens
3. **Generate new token** を押し以下を設定:

| 項目 | 設定値 |
|---|---|
| Token name | `UserListBot-readonly` など |
| Expiration | 1年など |
| Repository access | Only select repositories → このリポジトリのみ |
| Permissions → Contents | **Read-only** |
| その他すべて | No access |

4. **Generate token** → `github_pat_...` をコピーして保管

---

## 管理者の更新手順（リリースのたびに）

```
# 1. コードを編集する

# 2. config.py のバージョンを上げる（例: 2.0.1）
#    APP_VERSION = "2.0.1"

# 3. manifest.json を再生成
python tools/build_manifest.py --version 2.0.1

# 4. コミット＆プッシュ
git add .
git commit -m "Release v2.0.1"
git push
```

以上で完了です。次回従業員がアプリを起動すると自動でダイアログが表示されます。

---

## 従業員の設定手順（初回のみ）

1. アプリの「⚙ 設定」を開く
2. **自動アップデート（GitHub）** セクションに入力:

| 項目 | 例 |
|---|---|
| リポジトリ | `tsuyoshi-miki/userlist-bot` |
| ブランチ | `main` |
| アクセストークン | 管理者から受け取った `github_pat_...` |

3. **接続テスト** ボタンを押して「接続成功」を確認
4. **保存** ボタンを押す

以降はアプリ起動のたびに自動でチェックされます。

---

## アップデートの流れ（従業員側）

```
アプリ起動
  ↓
GitHub から manifest.json を取得
  ↓（新バージョンがある場合）
「アップデートがあります v2.0.1 → 今すぐ更新しますか？」
  ↓ はい
変更されたファイルのみダウンロード（差分のみ、高速）
  ↓
自動で再起動（更新完了）
```

---

## 注意事項

- トークンは各自の端末の `%LOCALAPPDATA%\UserListBot\settings.json` に保存されます（アプリ外には漏れません）
- インターネット接続が必要です（オフライン時はスキップされて通常起動します）
- トークンの有効期限が切れた場合は再発行して設定画面から更新してください
