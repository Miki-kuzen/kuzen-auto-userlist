# Slack 通知の設定

処理完了・エラー発生時に Slack へ自動通知できます。  
設定には **Webhook URL** の取得が必要です（Slack のワークスペース管理者が行います）。

---

## Webhook URL の取得方法

### ステップ 1 — Slack App を作成する

1. ブラウザで [https://api.slack.com/apps](https://api.slack.com/apps) を開く
2. 右上の **「Create New App」** をクリック
3. **「From scratch」** を選択
4. App Name に任意の名前（例: `UserListBot`）を入力し、通知を送りたいワークスペースを選択して **「Create App」**

### ステップ 2 — Incoming Webhook を有効化する

1. 左メニューの **「Incoming Webhooks」** をクリック
2. 右上のトグルを **「On」** に切り替える
3. ページ下部の **「Add New Webhook to Workspace」** をクリック
4. 通知を送りたいチャンネル（例: `#自動化通知`）を選択して **「許可する」**
5. 生成された **Webhook URL** をコピーする

```
例: https://hooks.slack.com/services/XXXXXXXXX/YYYYYYYYYYY/zzzzzzzzzzzzzzzzzzzzzzzz
```

> URL は他人に見せないでください。このURLを知っている人は誰でもそのチャンネルにメッセージを送れます。

---

## アプリへの設定手順

1. アプリの左サイドバーから **「⚙ 設定」** を開く
2. 「Webhook URL」欄にコピーした URL を貼り付ける
3. 通知のタイミングを選択する（下記参照）
4. **「保存」** ボタンをクリック
5. **「テスト送信」** ボタンで Slack にメッセージが届くか確認する

---

## 通知のタイミング設定

| トグル | 内容 |
|---|---|
| 全件完了時に通知 | すべての案件の処理が終わったときに成功・失敗件数を通知 |
| エラー発生時に通知 | いずれかの案件でエラーが起きたときに即時通知 |
| 案件ごとに通知 | 1案件が完了するたびに通知（件数が多い場合は大量通知になるためオフ推奨） |

---

## 通知メッセージの例

**全件完了時:**
```
✅ 転写完了: 成功 8件 / 失敗 0件
```

**エラー発生時:**
```
❌ エラー: TAPP — CSVのダウンロードに失敗しました
```
